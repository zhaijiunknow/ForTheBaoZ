# 项目完成度梳理

## 说明
本文档用于梳理当前 Unity 项目中已经完成、部分完成、以及尚未完成的内容。

本次判断基于当前仓库中的场景、脚本、配置与资源做静态分析，重点结合以下实现线索：
- 菜单/UI 与启动流程
- 战斗原型与推力对抗
- 槽位、符号、序列/QTE
- Roguelike 事件与内容承载
- 《绝味：理智蒸发》题材相关包装与叙事

### 4. 槽位系统基础原型已完成
当前项目不是没有槽位，而是已经存在可运行的基础槽位结构。

已确认内容：
- `MySkillManager` 持有 `skillcount` 与 `myskillbox`
- `BattleUI.SetGameButton()` 会根据 `skillcount` 动态创建按钮
- 按钮内容从 `myskillbox` 中读取，并映射到 `items` 列表
- Roguelike 事件可以增加槽位上限

说明：
- 这意味着项目已经具备“初始若干槽位 + 运行时动态生成 + 局内扩容”的基础结构。
- 现阶段槽位表现更像“技能/道具承载位”，而不是彻底抽象化的认知容器。

关键文件：
- `Assets/Script/Main/MySkillManager.cs`
- `Assets/Script/DanMuMain/GameSystems/Battle/BattleUI.cs`
- `Assets/Script/Main/RoguelikeManager.cs`

### 5. 符号层已经进入配置系统
项目中已经存在“符号”的程序表达，不只是概念层。

已确认内容：
- `skilltype` 枚举存在，当前包含 `None / qte / a / b / c`
- `ItemConfig` 中存在 `goodtype` 字段，用于给道具配置对应的符号类型
- 点击道具时，会把 `goodtype` 写入序列队列

说明：
- 从结构上看，这已经是“符号”的基础实现。

关键文件：

## 二、部分完成

### 1. 槽位 / 符号 / 序列三层骨架已经有，但还没有完全统一成“认知解构系统”
当前项目已经有：
- 槽位
- 符号枚举
- 序列判定

但目前更像：
- 技能/道具位
- 技术型符号枚举
- QTE 连段触发

还没有完全表现成策划中的：
- 槽位本身无功能，只承载认知单位
- 玩家重写符号与槽位关系
- 配方链条成为构筑核心
- 构筑行为本身改变“发疯节奏”

结论：
- 骨架已存在
- 表达层与系统整合层仍未完全完成

### 2. 文本系统有承载能力，但叙事还未形成体系
项目里已经有：
- 菜单文本
- 本地化 CSV
- 文本测试场景
- 飘字相关系统

但还没有明显看到：
- 成体系的剧情文本
- 角色对白库
- 事件叙事库
- 随理智变化而扭曲的文字表现

关键文件：
- `Assets/StreamingAssets/titles.csv`
- `Assets/StreamingAssets/options.csv`
- `Assets/Scenes/TextExample.unity`
- `Assets/Scenes/TextTestRoom.unity`

### 3. Roguelike 更像随机事件，而不是完整节点地图
目前已有的是：
- 单次事件
- 若干选项
- 立即生效的奖励

但还没看到：
- 路线图
- 层级推进
- 节点类型分布
- 关卡间地图流程

结论：
- 已有事件结构
- 未见完整路线/节点图系统

### 4. 项目已有玩法原型，但题材包装很弱
已经能看到一些玩法内容，但命名仍较原型化，例如：
- `Item1.asset`
- `Item2.asset`
- `qteItem1.asset`
- `rogue1.asset`

说明：
- 系统先行，内容包装滞后。
- 这类资源可以继续复用，但离正式项目表达还差一层整理。

## 三、尚未完成或暂未看到明确实现

### 1. 《绝味：理智蒸发》的题材包装尚未正式落地
目前没有明确看到：
- 项目标题正式替换
- 世界观文档
- 主角设定
- 包子铺相关文本
- 现实/幻觉冲突的系统化包装

一个明显信号是：
- `Assets/StreamingAssets/titles.csv` 里仍存在 `Your Game Title` 占位内容

### 2. 天平机制尚未看到明确实现
虽然当前推力对抗在抽象上已经有“拉锯感”，但还没有看到明确的：
- 天平对象
- 左右承重可视化
- 高度即位能的表现层
- 飞升 / 坠落围绕天平结构展开的正式系统

结论：
- 有可复用的数值对抗原型
- 但“天平化表现”尚未落地

### 3. 认知重构层尚未完整实现
目前没有明确看到以下能力已经存在：
- 玩家主动重写符号与槽位映射
- 通过局间节点永久改变配方需求
- 删除、克隆、替换槽位符号形成系统性构筑
- 复杂行为被逐步改造成肌肉记忆链

结论：
- 基础结构有
- 高级构筑和认知扭曲层尚未完成

### 4. 正式叙事系统尚未形成
未明确看到：
- 对话树
- 剧情节点推进
- 章节结构
- 失败文本 / 飞升文本 / 记忆碎片文本库
- 与理智状态强绑定的文字攻击性变化

### 5. 资源命名与工程整理尚未收口
目前仍能看到一些原型阶段痕迹：
- `Assets/Script` 与 `Assets/Scripts` 并存
- `DanMuExamole.unity` 命名像测试场景
- 多个资源命名为 `Item1 / rogue1 / qteItem1`
- Build Settings 仍引用缺失场景 `Assets/Scenes/SampleScene1.unity`

说明：
- 项目已经不是起步阶段
- 但也还没有进入完整收尾和规范化阶段

## 四、总结

### 当前最准确的判断
这个项目**不是从零开始**，也不是只有空菜单模板。

它已经完成了以下核心骨架：
- 菜单/UI 框架
- 输入系统接线
- 战斗原型
- 推力对抗
- 槽位动态创建
- 符号枚举配置
- 序列/QTE 判定
- 道具体系
- 简易 Roguelike 事件
- 基础 AI 对手

但它**还没有完整完成**《绝味：理智蒸发》当前策划中最有辨识度的那层内容：
- 天平化表现
- 认知解构式槽位重构
- 正式题材包装
- 完整叙事残响
- 完整 Roguelike 节点推进

### 一句话概括
当前项目更像是：

> **已经有“槽位 + 符号 + 序列 + 推力对抗”的可运行原型骨架，但尚未完全升格为《绝味：理智蒸发》这套明确主题、强叙事、强表现的正式版本。**
